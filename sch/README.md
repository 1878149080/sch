## git 提交规范
✨ feat - 新功能 feature  
🐛 fix - 修复 bug   
📝 docs - 文档注释  
🎨 style - 代码格式(不影响代码运行的变动)  
🔨 refactor - 重构、优化(既不增加新功能，也不是修复bug)  
⚡️ perf - 性能优化  
✅ test - 增加测试  
🔧 chore - 构建过程或辅助工具的变动  
⏪ revert - 回退  
🚀 build - 打包  
示例1：  
1.`[fix]` 修复xxxBug  
2.`[feat]` 新增xxx功能   
3.`[perf]` 优化xxx代码   
示例2：  
1.🐛fix(菜单名称) 修复了xxxBug  
2.✨feat(菜单名称) 新增xxx功能   
3.⚡️perf(菜单名称) 优化xxx功能
